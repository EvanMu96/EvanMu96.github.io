N 的范围 [1, 30].

[消费电子] Apple的T2芯片究竟是做什么的

2020-01-10

前言

我的第一台苹果电脑是2011款的macbook pro，这台电脑当时的屏幕素质完全震撼了当时在上高中的我。在这台mbp2011之前，我用的是一款acer的14寸塑料壳笔记本，用来玩会声会影剪辑视频实在是勉勉强强。
2018年，我来到香港读书。由于学校和信港电脑合作有价格实惠的macbook学生机，我再次换掉手上的win10笔记本把主力笔记本切换到了苹果的macbook pro 2018款。我拿到手之后发现，新的mac多了一个coprocessor，它的名字叫做T2 security chip，是一块。如果我没猜错的话，这枚coprocessor是用来加强加密功能提升系统安全性的。之前我对类似的全盘加密机制也不怎么了解，买过的windows笔记本也从来没有TPM，因此去翻了一下T2的技术手册对照着自己学过一个学期的密码学只是梳理一下。

大纲

  • Apple T系列芯片的历史
  • T2芯片用于File System加密
  • T2芯片用于系统Secure Boot
  • T2芯片用于touch-id指纹认证
  • T2芯片用于保护麦克风使用权限
  • Reference

Apple T系列的历史

在现代的计算机加密方法中，非常常用的一种加密方法是对整个硬盘做加密。对加密的实现可以用执行在CPU上的内核级程序也可以使用一个独立单一功能的硬件完成。相对于使用软件方法，独立的加密芯片硬件有着得天独厚的优势。首先，独立的硬件保证了未经加密的raw data不会进入cpu中，确保了骇客不可以hack进系统内核态读取未经加密过的数据。其次，因为CPU不再负责加密，独立芯片的方案在IO和CPU利用里上有了比较明显的效率提升。以当前2020年来讲，现代的操作系统都会提供独立加密芯片的支持。比如Windows操作系统的Bitlocker，可以搭配TPM硬件完成这项功能。而mac的这项功能由FireVault软件和T系列的芯片提供。

Apple从搭载touchbar的第一代macbook pro开始就使用了T1这枚ARMv7的SoC，主要用于运行touchbar的bridgeOS，并独立负责touch id指纹hash的存储，以及保护mic不会被黑客劫持。而在2017年的iMac Pro上，Apple开始推广自己的T系列加密芯片的第二代。这一次，大容量的SSD的加密也将交给专门为AES encryption/decryption设计过的64bit ARMv8芯片。无论是功能的多寡还是性能，T2在纯粹的性能上可谓是远远超过了它的前辈。也顺理成章地将FireVault加密与解密的运算职责挪到了T这块coprocessor上。

T2芯片用于File System加密

![image-20200113211150372](/images/Screenshot 2020-02-12 at 12.53.06 AM.png)

Apple T2芯片使用DMA path访问SSD和内存。而在加密方式上，T2内部的专为AES加密算法设计的AES Cypto Engine。AES算法是一种block cipher，其加密方式是将数据分成固定的块一块一块（block）做对称加密操作。这里提到了一个术语叫做对称加密，既然有对称加密，那肯定也有非对称加密，这里我们暂且不展开讲非对称加密的原理。只说一个key point就是非对称加密在数学原理上使用的是one-way function。这种函数易于正向计算而在数据规模很复杂的情况下不容易反向推出原始数据，加密和解密也并非是互逆的操作。而对称加密不同，加密和解密是完全互逆的操作。也正因为对称加密是互逆的，对实时性强，IO量大的数据加密我们会优先使用对称加密操作。即使是https这种利用了非对称加密的技术，也会在建立secure channel之后切换到对称加密模式提高性能，这种方式叫做混合加密。

说远了，说回T2芯片使用的AES-256（256代表key的长度）。这样的key有两个，一个是设备唯一的UID，另一个是出厂时的Group ID(GID)。他们共同为AES Crypto Engine提供key输入。对于这两段key的存储，这两段key被号称完全锁死在Secure Enclave Processor里（iPhone的A系列SoC中也设计了这样的芯片[6]），甚至不可以被硬件级debug interface获取到。看来在设计上，苹果对安全性做了破釜沉舟的加密设计。同时，为了防止加密在苹果的T2官方白皮书[1]里有写到，This dedicated   AES engine makes available only the results of encryption or decryption operations it performs。也就是说AES Crypto Engine是没有内存的，大概率是一块dedicate ASIC。

众所周知，计算机系统对key的管理是有层次的(hierarchical)。就仿佛在现实生活中，我们都会把保密性稍弱的钥匙串放在保密性更强的保险柜里。保险柜的钥匙是所有钥匙的Master key，一旦保险柜被攻破，那么所有的其他的保密措施便形同虚设。FireVault作为对SSD全盘加密的主体，在key的层次上，一台macbook pro的最顶层的key是你设置的password，然后才是UID和GID。

![image-20200114005224961](/images/Screenshot 2020-02-12 at 12.53.15 AM.png)

因此，对于一台带有T2芯片的mac，我们在保证我们的system password没有被社工方式获取的前提下，无法通过任何方式获取硬盘中的内容。

T2芯片用于系统Secure Boot

如果读者尝试过自己给电脑重装操作系统，我们不难发现我们是从BIOS中引导其他的操作系统或者进入recover mode就可以抹除系统盘并且重装操作系统的。 或者说，我们可以从更先前的步骤做手脚，修改non-volatile memory中的UEFI古固件。

所以无论是什么操作系统的计算机，都应该在启动阶段尽可能的早杜绝不明来源的二进制程序的读取。对计算机启动过程感兴趣的读者可以阅读[5]作为补充，在这里便不做叙述。

T2芯片的secure enclave中会存储可信的UEFI固件的signature和操作系统的signature。具体secure boot过程中对signature的验证步骤见下图。

![image-20200119002813239](/images/Screenshot 2020-02-12 at 12.53.23 AM.png)

有了这样的防护措施，哪怕是骇客想启动recover mode或者引导其他的操作系统也会被T2芯片拦截。

T2芯片用于touch-id指纹认证

指纹作为重要的生物特征，将这个重要的敏感信息存储到操作系统内任意的存储位置都有风险。尤其是苹果想把touch-id这个功能放到笔记本电脑上，为了防止被hack也要额外的小心。

不过对于这一部分，mac的解决方案和A系列SoC的方式是差不多的，很明显是借鉴了多年来移动SoC设计中的best practice。

T2芯片用于保护麦克风使用权限

很早之前曾救有一张图在互联网上传播，就是马克 扎克伯格的笔记本电脑用贴纸贴住摄像头和麦克风的一张图。不过现在不用了，T2在白皮书中提到，当盖子合上之后，麦克风是相当于硬件断开的。不过众所周知，从macOS Mojave开始，OS已经对麦克风权限在软件层面上做了比较严格的权限。这次再硬件上的保护可以说把“用贴纸贴住麦克风”这件事成为了历史。

至于视频，T2没有对视频做断开。主要原因还是盖子合上的时候原生的笔记本摄像头已经什么都看不到了。（当然如果你是一个视频主播，或者有额外的摄像头，从安全角度我还是建议对摄像头做额外的措施）。

新博客的第一篇文章就是补充一篇上一次重启博客没有完成的学习笔记。笔记里的很多内容是参考了吴军博士的《数学之美》，维基百科和知乎上的解答，在这里感谢他们对P/NP这个未解之谜的解释。
下面这段文字来自《数学之美》（作者：吴军博士）的附录，偏向于inituition。
计算机的算法效率是用计算复杂度(Computational Complexity)来衡量的。计算的时间显然和问题的大小有关。比如对10000个实数排序和对1000000个实数排序所用的时间是显然不同的。问题的大小在衡量计算时间复杂度时是变量，一般用N来表示。而计算量是N的一个函数f(N)。这个函数的边界可以用数学上的大O概念来限制。如果两个函数f(N)和g(N)在大O概念上相同，也就是说当N趋近于无穷大时，他们的比值只差一个常数。比如f(N)=Nlog(N), g(N)=100log(n)，他们就被看做是同一个数量级的。同样，如果两个计算机算法的计算在大O概念下相同，只差一个常数，我们认为它们的计算复杂度相同。计算的复杂度关键看函数而不是这个常数。
如果一个算法的计算量是N的多项式函数(Polynomial Function)，则认为这个算法是多项式函数复杂度的。如果一个问题存在一个多项式函数复杂度的算法，则这个问题成为P问题。如果计算量比N的多项式函数还要高，虽然理论上讲如果有无限的时间也是可以计算的（图灵机概念的可计算），但实际上市不可计算的。这时称为非多项式(NP)问题。比如找到每一步围棋的最佳走法就是一个NP问题。
如果一个NP问题，虽然找不到多项式函数复杂度的算法，但是对于一个算法可以再多项式函数复杂度的时间内证实这个方法正确与否。那么这个问题成为NP-Complete问题。如果一个问题，它的计算复杂度至少是NP-Complete，那么它被称作NP-Hard问题。换句话讲NPC问题是NP-Hard问题的一个子集。

以下的文字来源于我对维基百科和知乎上的文章的整理，也补充上了NP-C Problem的介绍。
P类问题：所有可以在多项式时间内求解的判定问题（返回一个yes/no）构成P类问题。比如验证一个数是否是质数，我们可以从2一直除到这个数本身来验证这个数有没有除了1和本身以外的factor。
NP类问题：所有非确定性多项式时间可解的判定问题构成NP类问题。比如在正整数集合中找到下一个质数。求解这个问题是不可以按部就班的解出来的，只能够靠猜或者是线性搜索。然而，验证一个数是否是质数是一个P问题。因此我们就有了下一个定义
NP-C问题：NP中的某些问题的复杂性与整个类的复杂性相关联。这些问题中任何一个如果存在多项式时间的算法，那么所有的问题都是多项式时间可解的。这些问题被称为NP-Complete问题。（即可以转换到一个关联的P问题）
人们发现，所有的完全多项式非确定性问题，都可以转换为一类满足性（验证）问题的逻辑运算问题。既然这类问题的所有可能答案都可以在多项时间内计算，人们就猜想，是否这类问题存在一个确定性的算法，可以在多项式时间内直接搜索出正确的答案，也就是所以P = NP这个假设是否成立呢？
举个例子，神经网络的optimization就是一个NP问题，我们可以用迭代的gradient descent不断向更优的方向逼近。然后用validation set验证结果的好坏。

Introduction to Reinforcement Learning

Old paradigm vs. New Paradigm

In the tradational computer science method, we need to program every action of computer. Because the progressive research of Machine Learning field, we can provide some examples to computer, making it learn to accomplish a task based on examples. That is generally called supervised (machine) learning.
However, to let machines learn a task need a lot of examples or data. We could label all of the data but that is so laborious. Alternatively, unsupervised learning or semi-supervised learning is more realistic and smart for huge mount of data. There is another method which neither supervised nor unspuervised, the reinforcement learning.

What is Reinforcement Learning?

Some well-known nickname, AKA

  • Optimal control
  • Approximate dynamic programming
  • Neuro-dynamic programming

Here is the definition from Wikipedia, you can read this paragrahph for a general view.
Reinforcement learning is an area of machine learning inspired by behavioural psychology, concerned with how software agents ought to take actions in an environment so as to maximize some notion of cumulative reward.
Just like the artifical neural network borrowed the concept of neuron. The reiforcement learning is alsor inspired from creatures.
If you want to train a dog to do some specific reactions, you need to reward it when the dog did the correct reactions, for example pleasure and food. On the opposite, some negative reward should be applied when the dog did bad. That is the basic idea to do reforcement for training a dog. If we do the similar things to a machine, that is so called reiforcement learning!

The components in Reiforcement Learning Problem

We need to train an agent instead of a dog for a general reinforcement learning task. So the first component should be the Agent. Except the agent, the all of other things are enviorment of the agent. The agent can change the state of environment by actions, then the environment feedback reward to agent, which is what a reinforcenemt learning system do.
